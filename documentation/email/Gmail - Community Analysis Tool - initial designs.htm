Puneet,

It would be great if we could meet once a week on Wednesdays at 4:30 Central / 2:30 Pacific (starting next week). Currently, our group will be experimenting with scraping in Java and PHP. We will consider the pros and cons of each one and eventually have to consolidate on one of these languages by next week.

My personal preference is to use PHP because it is meant to work with the web. It has powerful built in scraping functions, which will allow us to make the code simple and readable. If we were to do this in Java, we may need to import dozens of libraries and take many lines of code to do something PHP could do in one line.  I'm assuming that Java will run faster than PHP because PHP is completely interpreted, whereas Java is "compiled" (sort of) and has better multi-threading capabilities.  But that really isn't an issue because 90% of the scraper's run time is taken up by downloading pages, which is not dependent on the language used.  If during our initial development, we find that we really need efficiency, we would switch to Java.
