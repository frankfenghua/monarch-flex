Based on what we talked about with Puneet and Vera, here are some workflows that I came up with:

1. Sign up for service
2. Create community and create website in that community
3. Create website within an existing community and attempt to view the keyword statistics of that site (there should not yet be any)
4. Start adding a website but return to My Communities view before submitting.
5. View count stats for keyword x in community y
6. Create a website and view a sample adapter while starting the RegexEditor
7. Change the frequency of a scrape of one of your websites
8. Add a keyword to a community.

Feel free to add to the list or remove/modify anything that our app will not be able to do. From our feature list, it seems like it should make it possible to do all of these. I think we should finish this list and send it to Puneet and Vera tomorrow.
