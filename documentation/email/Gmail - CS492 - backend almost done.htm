Installation instructions:
  1. Install phpMyAdmin so you don't have to do all these following steps with command line. It preinstalls for you in WAMP.
  2. Create a new DB named "threadless". Import database/threadless.txt on it.
  3. Open database/master.txt and change the last line with the DB name, username, password, host (localhost), and unix timestamp (just run <?php echo time(); ?> to find that out).
  4. Create a new DB named "communityanalysis". Import database/master.txt, which you just edited, on it.
  5. Open database.php and edit the lines [24 - 25] with your MySQL username and password. 
  6. Run index.php to populate the threadless DB.
The program only populate's the thread table of the threadless DB, because no one answered my question about how Threadless' link structure works (I mean I know how it works but don't know why are they doing this). I also asked on an HTML forum and no one replied.

Description of the system:
  • Plugins for sites you write should be stored in folder "plugins". Copy the structure of plugins/threadless.php to create a new plugin. You must name your file with the base domain of the site minus the suffix and http://www part.
  • We have one master database called "communityanalysis" that stores information on how to connect to the specific sites' databases. This is not useful when you're developing on your home PC, but will be useful when we move to a real web host, because web hosts determine new username, password, and server location everytime you create a new database. Unless we get a $99 / mo dedicated server which we can have total control over a we would have over a "home PC", we can forget about this master database. But I highly doubt we will get anything good. 
  • Each specific site's database must contain these following basic tables. More tables can be added for specific functions of the site. For example, Threadless could have a "t-shirt designs" table addon.
    • keywords
    • links
    • posts
    • threads
    • users
  • I have an incomplete scrape.php class which only scrapes the outer threads right now. It will eventually go into the thread and scrape the replies.
  • We will need to create an analyze.php, which computes averages, standard deviation, and other various stats from the the info that scrape.php injects into the database.
  • index.php is analogous to main of a C language. Currently, it has no preloader bar thing, so just watch the browser circly GIF rotate to see when it's done working.
