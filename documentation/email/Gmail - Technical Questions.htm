Hi Vera,

These are the two main questions we had from our last meeting:

(1)
There is an issue with plotting multiple subsequent line series within the line chart component. Attached is a snippet of code I use for plotting multiple lines. The noticed behavior is that some combinations of lines plot together fine but others don't. For example, if I have line series A, B, C, and D which all contain their own set of data points, and I push each of them onto an array collection and then proceed to set the lineChart.series attribute equal to the array collection, I might observe the following behavior : if all lines are selected then A is the only line which is displayed until B is deselected, in which case C might show up with A. Furthermore, if C is deselected then line D might show up with A.
The bottom line is that A,B,C, and D never display simultaneously.

(2)
For real-time highlighting of regular expression matches in a TextArea we have adapted code from the FlexLib Highlighter class (link). Our code extracts the underlying TextField of a TextArea, draws a transparent bitmap on top, and draws rectangles on the bitmap corresponding to visible matches. In attempting to adapt the same technique for HTML we have added a bitmap as the child of an mx:HTML component but highlights do not show correctly. Rectangles are found using the location and offset properties of the matching HTML elements. The questions are: should this be possible? Can you think of another approach that makes more sense? There seem to be issues with drawing on top of rendered HTML for some reason. Our original approach for HTML highlighting modified the styles of the HTMLElement objects in the page but this did not seem to perform well in real-time. In a search, all examples are using javascript to modify the object model of the page rather than drawing on a layer above the page.
