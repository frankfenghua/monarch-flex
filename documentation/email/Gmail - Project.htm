Requirements Document v0.1

What is it?

Our project is a Community Analysis Tool that will analyze online communities like forums, blogs, and news sites, and provide a user with a way of viewing the resulting data. We are completing this project for our liaison at Adobe, Puneet Goel, within the context of the course, CS 492/493: Senior Project in CS, at the University of Illinois. 
Our deliverable will be the source code and any associated documentation that we have generated. Specifically, the project will include a server-side portion which is responsible for spidering and populating a database with relevant information and a separate client-side portion which may run on a different machine. The client will be used by a user to communicate with the database server in order to retrieve community analysis results. Some of the data analysis may also be done on the client side. Upon our completion of a prototype system, the goal is to open-source the project.

What is needed?

The only requirement up to this point is that we create an application that uses Adobe Flex and Adobe AIR technologies to display results of web community analysis. This requirement implies that we also must create a program for collecting and analyzing community data and a way of serving out this data. Additional concrete needs have yet to be laid down.

What is wanted?

On the user side, this application will provide various forms of data visualization, like plots and pie graphs, in an attractive desktop application. Viewable statistics may include things like community activity surrounding a given keyword, popular keywords, the number of posts per day, posting activity of users, and various other trends that may be assessed through page source analysis.

What has already been done?

Our liaison at Adobe has stated that this is a project in its early infancy so there is no existing source. However, project advisors suggested that we look into Google and Yahoo open-source technologies as possible starting points for implementation of the web-crawling capabilities required in our program. As another consequence of the recent conception of our project, our team will be responsible for drafting a design document from scratch that includes a list of required features and desired features.

Why?

There are two motivations for building this community analysis tool:
 (1) Puneet, the product marketing manager of Flex and Air, wants his products to be used in universities. He hopes that we can demonstrate the utility and power of these products in our application.
 (2) This tool could be useful for several types of users, both within Adobe and outside of Adobe. For examples, researchers could use it to understand human behavior on the web or marketing executives could use it to find out what hot products people are most interested in. In addition to these, there are a wide range of scenarios in which a community analysis tool could be applied.

What tools (hardware or software) do we require?

Adobe was not willing to provide their Master Collection to us for the completion of this project, nor were they certain that they could provide us a dedicated server. We will need one or more fairly powerful machines to do web-crawling tasks periodically and serve many requests very quickly.

Limitations

Our tool will not be able to analyze cross-community interdependencies involving specific users. It is too difficult to determine whether two accounts on two forums are the same person. For example, John Smith could have the screen name LeetHaxor on a computer forum while he could have the screen name Dominator999 on an MMORPG forum. We would not be able to aggregate the stats of this person across both forums.
Because the web is so free form, every community will have differing HTML structure. For example, one news site could write its titles as <h1><b>Free</b> Adobe Master Collection</h1>, whereas another news site could write theirs as <div class="title">Free Adobe Master Collection</div>. It would be nearly impossible to write a single algorithm to parse any website given to us. There are just too many coding variations. Instead, we will develop a plug-in system by which the structure of a website may be described. Using a plug-in developed for a specific site, our server will be able to collect data from the site and populate its database correctly. In the future, if websites want to be included into our analysis tool, they could supply their own plug-in or these plug-ins may also be supplied by users.
